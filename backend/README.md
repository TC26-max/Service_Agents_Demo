# Nimbus RAG backend (full semantic) — Vercel

A tiny always-on API that does **real semantic retrieval** (OpenAI embeddings + cosine) over
the knowledge base and returns a grounded, cited answer. The model key stays server-side, so
the public page needs no key.

```
backend/
├── api/chat.js          # POST /api/chat  → { answer, sources[], faithful }
├── ingest.mjs           # one-time: build kb-embeddings.json from kb.json
├── kb.json              # the knowledge base (copied from the project root)
├── kb-embeddings.json   # generated by ingest (commit it)
├── vercel.json
└── package.json
```

## Deploy (≈5 min, always-on, free tier)

1. **(Optional) Pre-embed for faster cold starts.** The backend builds embeddings on its
   first request automatically, so you can skip this. To pre-bake: `OPENAI_API_KEY=sk-... npm run ingest`.
2. **Deploy to Vercel:** push this `backend/` folder to a repo and import it at vercel.com
   (or run `npx vercel`). In **Settings → Environment Variables**, add:
   - `API_KEY` = your key (required)
   - `PROVIDER` = `mistral`, `gemini`, or `openai` (default `openai`)
   - optional: `CHAT_MODEL`, `EMBED_MODEL`, `API_BASE`, `ALLOW_ORIGIN` (your site origin; default `*`)

   **Free options:** `gemini` (sustained free tier, includes embeddings) or `mistral` (free
   Experiment tier, model `mistral-embed` + `mistral-small-latest`). `openai` is paid but
   pennies. All three use the same OpenAI-compatible shape, so only the env vars change.
3. You'll get a URL like `https://nimbus-rag.vercel.app`. Test it:
   ```bash
   curl -s https://nimbus-rag.vercel.app/api/chat -H "Content-Type: application/json" \
     -d '{"bot":"support","messages":[{"role":"user","content":"What is your refund policy?"}]}'
   ```
4. **Point the page at it:** in `../index.html`, set `const BACKEND = "https://nimbus-rag.vercel.app";`
   The hub then uses full semantic RAG, and every visitor gets grounded, cited answers with no key.

## Notes
- **Cloudflare alternative:** the same handler maps to a Cloudflare Pages Function /
  Worker with minimal changes (use `env.OPENAI_API_KEY`); both are always-on with no cold-start sleep.
- Swap the in-memory cosine search for pgvector / Pinecone when the corpus grows; the request/response
  shape stays the same.
- This is the "full semantic" deployment path. The page also ships with on-page keyword
  retrieval so the demo works even before this backend exists.
